之前向大家分享过一篇开通 Claude、ChatGPT 会员的文章，主要内容写的是如何通过 App Store 的应用内购套餐，来获取稳定的官方订阅，优势是不需要任何的信用卡或者支付方式，只需要去购买对应区域的礼品卡即可。

如果你还没看过，可以在这里看详细的开通教程：[没有海外信用卡也能稳定订阅 ChatGPT/Claude](https://www.yizhe.me/zh/blog/202603/app-store-ai-subscription/)

今天主要分享一下我自己使用 Claude Pro 以及 Max 的日常经验。

> 其实最开始我一直在纠结要不要去写这个东西，因为关于 Claude 的账号注册、保号可以在 Google 或者 X 上找到一大堆参考材料。所以我也很纠结，我再去写这个「教程」，到底意义有多大……
>
> 但是其实我看过绝大多数的 Claude 保号和使用的教程文章，我觉得我的使用方式，跟他们分享的还是有很大出入的，所以我觉得不想把我的这个分享叫做教程，只能算是我分享一下我的使用经验吧。

如果你看过其他的账号使用/保号教程，你可能大概也都看过：「住宅 IP」、「**指纹浏览器**」、「U 卡」等这些概念了。但是我要说一下，我的使用经验里，没有用到这里的任何一个。

## 账号概况

先说一下我的账号数量，我前前后后注册过 5 个 Claude 账号，其中只有一个是通过邮箱+手机验证码注册的，其他的全都是直接通过 Google 账号登录的。然后到现在为止，账号还在的有三个。其中两个被封号的我后面会说明具体原因。

还有一点我想说明一下，你在网上看到的所有的关于 Claude 封号的分享，可能都不一定是准确的，一切还是需要以你的使用经验为主。同样的方法，在别人那里不会被封号，但是有可能到了另外一个人的手里就会被封号。而 Claude 官方又从来没有公布过何种使用方式会导致账号的封禁，因此一切都是网友自己使用经验的推测。

字节的这篇文章有详细的从代码层面的分析说明，是一个非常有价值的对封号机制的深度报告，感兴趣可以看一下。报告里的内容至少是直接从 Claude Code 的「开源」代码里找到的证据，所以要比其他的用户推测可靠很多：[Claude Code 封号机制深度分析](https://bytedance.sg.larkoffice.com/docx/QNsIdi8VFoKm00xY8SRlkXxygve)

## 注册经历

我的第一个 Claude 账号注册时间大概是 2025 年年初了，当时因为我刚把 Outlook 邮箱设置了别名，那一段时间，注册很多服务都是通过 Outlook 注册的。自然注册 Claude 也是一样的。当时直接使用邮箱注册，还需要接收验证码，然后我在前几次注册的时候，一直没有合适的手机号接收验证码（Claude 不给我的两个菲律宾手机号发验证码），然后就反复卡在这个地方，最终导致我一直向 Claude 请求发送验证码，并且是隔两三天请求几次，然后就这样尝试了几次之后，我的这个账号就直接被封禁了……

现象是当我最后一次去尝试的时候，页面直接弹出封号提示。所以我的第一个账号，就在我还没有注册成功的时候，就被封掉了，就是这么迅速。

![Claude 封号提示](./claude-banned.png)

后来我在注册第二个账号的时候，提前在淘宝买了接收验证码的海外手机号，花 10 块钱完成了验证码接收，就通过我的谷歌邮箱注册成功了。开始在这个账号下进行充值 Pro 套餐的使用。

这个账号就这么相安无事到现在，都还在正常使用。

当然在我的使用过程中，一个 Pro 套餐是根本不够用的。有时候，分析一个问题，5 小时的限额就直接消耗完了。所以基本上属于完全不够用的状态。

这种情况下，我就开始开通第三个和第四个账号了。

后面的账号，我就没通过邮箱+手机号验证码的形式去注册了，而是直接通过谷歌邮箱。直接点击上面截图里的 **Continue with Google** 去注册登录的。

后面我也发现也有很多人在通过 Google 账号登录时，可能是因为 IP 或浏览器的问题，即使使用了 Google 登录，系统还是会要求输入手机号并接收验证码。

不过至少对我来说，在我的这个 IP 和环境下，使用 Google 账号登录时并没有被要求提供验证码。

## 登录环境

这里也分享一下 Google Chrome 的使用经验，我觉得一定程度上完全可以替代上面说到的指纹浏览器了。

这个功能就是 Chrome 的个人资料，官方的教程在这里：[通过多份个人资料使用 Chrome](https://support.google.com/chrome/answer/2364824?hl=zh-Hans&co=GENIE.Platform%3DDesktop)

由于我的工作性质，经常要使用多套环境，有多个租户需要登录，所以总不能一直在一个浏览器里，始终通过退出账号再登录账号使用。所以很早就开始使用这个个人资料的功能了。下面是我的个人资料，这还不是最夸张的时候，最多的时候，一个屏幕都装不下，需要滚动才能找到对应的个人资料。

![Chrome 个人资料](./chrome-profile.png)

所以我在使用 Claude 账号的时候，多个账号之间，就是通过这个来管理的。也就是说直接为每一个账号创建自己的个人资料。无论是通过 Google 登录还是通过邮箱登录，都始终在同一个个人资料里，不会去跨资料使用。

我觉得这个方案已经足够了。这比重新下载一个所谓的指纹浏览器要轻量得多，并且对我来说，截止目前使用起来也毫无压力。没有遇到因为这个而封号的情况。

## IP 地址

我相信你之前已经了解过很多 Claude 的使用教程里，都会提示你一定要使用家宽 IP，提高 IP 质量。但是说实话，我现在还没有自己的家宽 IP 呢。

目前除了机场的 IP 节点之外，我有一台自己的 DMIT 美国服务器。这台服务器虽然搭建了对应的服务，但是实际上我也不怎么使用。大多数情况下，都是直接通过机场的 IP 去连接的 Claude、OpenAI 的服务。少部分时间，会使用自己的节点去访问。

![机场节点 IP 质量](./claude-ip.png)

我使用的这个机场，这 IP 质量应该是属于烂到家了吧哈哈，但是实际上我绝大多数是通过这些 IP 去使用的。截止目前，我的 4 个付费过的账号，没有因为网络或者 IP 原因被封号的。

所以我觉得网上说的因为 IP 质量被封号的，八成是没有真实证据的。关于这一点，还有一个佐证材料是，上面字节分析的 Claude 封号分析文章里，第六章里也没有提到过 IP 相关的证据。

## 付费方式

关于付费方式，在开头的文章里也说明了，我所有的 Claude 订阅，不管是 Pro 还是 Max 全都是通过 App Store 内购去订阅的，而且全都是通过尼日利亚地区订阅的，没有使用美区 App Store 也没有使用信用卡。

但是我确实是因为付费方式被封了第二个号。而且这个号已经正常使用一年的时间了，可惜没了。

当时的流程是这样的，这个账号一直稳定地通过尼日利亚内购来付费，但是因为那段时间大家都在流行开通 Bitget 的 U 卡，我也就顺势开通了一张。开通之后，当时有个活动，就是通过 U 卡支付 AI 会员，会返现 10 美刀。我想着也通过这个活动验证一下这个 U 卡是可以使用的。其实即使是返现 10 美刀，在价格上对我来说也并没有任何的吸引力。因为我日常的订购价格已经够便宜了，纯粹是为了测试这张 U 卡。

有意思的是，当时套餐订阅到期之后，我就直接在网页通过 U 卡支付了。支付成功之后，就开始高高兴兴地在 Claude Code CLI 里使用了。

![U 卡支付成功](./claude-pay.png)

结果，中午十二点付费，到了下午两点就直接封禁了。现象是 CLI 里直接提示 suspend 了。

本来以为我的费用不会再退还了，不过等了三天 20 美刀还是正常返回到了我的卡里。

对于 Anthropic 的封号速度，没得说；退款，也没得说。

至此，我的使用时间最长的一个 Claude 账号就这么没了。

但是后面几天，其实我在 X 上也看到有的人通过这个 U 卡也是能开通成功的，所以可能我的两个付费方式差别过大，才是被封禁的原因。所以后面的付费账号，我就不会再去变更付费方式了，始终保持尼日利亚内购付费。

而且我觉得最神奇的是，我通过同一个 App Store 账号去内购付费不同的 Claude 账号，也并没有任何的限制。最多的时候，同一个 App Store 账号去付费三个 Pro 账号，也并没有遇到被封号的情况。

当前我已经把其中一个 Pro 账号升级到了 Max，另外两个会在这个月之内陆陆续续到期。但是对我来说，那两个 Pro 账号已经基本上不怎么用了。

## 使用方式

关于我日常的使用方式，绝大多数时间，我都是在 Claude Code CLI 里进行使用的。当时还发现，很多人不知道 Pro 套餐也可以在 CLI 里使用。其实很早之前就可以了。

在使用多个 Pro 账号的阶段，每一个账号达到限额的时候，就通过 /login 切换账号。当然也有一些其他的多账号管理工具，例如 CC Switch。只不过对于我来说，还是使用 /login 去切换账号。

至于手机使用，我相信你也在网上看到过，很多人说不要在手机上登录你的 Claude 账号使用。

我想说的是，你都买了这么强的 AI 工具了，难道你还只限制在只能在一台设备上去使用吗？那也太可惜了吧。

所以我日常的使用是，不管我有几个账号，我首先会在我的 MacBook Pro 上登录，同时我也会在我的手机上登录。在一些情况下，我也会在服务器上登录。

在手机上，我使用最多的功能就是它的 Remote Control（远程控制）功能。

有时候，我在电脑上的工作没有完成，可以直接开一下 Remote Control，然后在手机上继续进行远程的工作。还有一些情况下，Claude 做的调研报告我觉得比 Gemini 的工具还要更好一些。这些工作全都是在我的手机上完成的。

甚至我现在的使用方式都默认变成了下面这样，默认打开绕过权限和远程控制：

```
claude --dangerously-skip-permissions --remote-control
```

![Claude Remote Control](./claude-remote.png)

当然，在我订阅 Max 套餐之前，多个 Pro 账号确实有一些不方便的地方。

我每次使用不同账号时，当我切换到账号 A，在我的命令行工具（CLI）中开启了一些远程工具，我的手机 Claude App 也必须切换到账号 A 才能进行对应的远程控制。这虽然非常正常，但总归有些不方便。不过自从我现在用了 Max 套餐之后，基本上就不再需要切换账号了。

## 网络规则

我在两台电脑和一台手机之间，有可能会频繁地去切换 Claude 账号。但是这里有一个前提：我的手机和电脑都使用 Surge 作为网络代理工具，并且它的代理配置文件是通过 iCloud 自动同步的。

也就是说，我在其中一台设备上修改了某些规则，这个时候其他设备是可以同时同步这一规则的。

所以即使我是在多台设备之间使用，它们日常使用的网络代理也基本上是相同的。不管是使用机场的 IP 还是使用我的 VPS，至少对于同一个账号而言，它的设备和 IP 都是基本稳定的，虽然这个 IP 可能有成千上万的人在同时使用。

我也可能会经常在这些设备上去切换 Claude 的账号。这个我上面已经说过了，因为在我使用多个 Pro 账号的时候，我的额度是不够用的，所以我需要去频繁切换。

这里也分享一下我的 Surge 网络规则，里边集成了一些常用的 AI 工具。其实对我来说，最主要、最重要的就是 Claude 的规则。你可以让各种 AI Agent 把这些规则翻译成你自己的代理软件所需要的格式：[Surge AI 规则列表](https://github.com/ennann/SurgeToolkit/blob/main/Rules/ai.list)

## 总结

从我的使用经验能看出来，我的使用方式和网上说的很多教程其实有很大的差别。

没有住宅 IP，没有指纹浏览器，没有美区信用卡——我用的是机场节点、Chrome 个人资料和尼日利亚 App Store 内购。两个被封的号，一个是因为注册阶段反复请求验证码，另一个是因为突然变更付费方式。剩下的三个账号，到现在都还在正常使用。

所以我觉得与其花精力去折腾那些复杂的环境伪装，不如把重点放在两个方面：**保持稳定**和**避免突变**。稳定的登录环境，稳定的付费方式，稳定的网络出口——这些才是真正让账号存活下来的关键。

当然，正如我在文章开头说的，所有关于封号的经验都只是个人推测，包括我这篇也是。Anthropic 从来没有公布过具体的封号标准。我能做的只是把自己的真实使用方式分享出来，供大家参考。
